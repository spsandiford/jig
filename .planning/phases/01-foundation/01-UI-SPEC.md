## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon padding, gap between toolbar buttons |
| sm | 8px | Compact element spacing, padding inside toolbar buttons, tab inner padding |
| md | 16px | Default panel padding, gap between tab label and icon |
| lg | 24px | Section padding, status bar vertical padding |
| xl | 32px | Layout gaps between major regions (editor panel, tree panel) |
| 2xl | 48px | Not used in Phase 1 (no major section breaks at this scale) |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Touch targets on toolbar buttons: minimum 36px height (developer desktop tool — relaxed from 44px mobile standard). Source: Claude's discretion.
- Status bar height: 28px fixed (matches VS Code status bar convention).
- Tab bar height: 36px fixed.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Status bar text, tree node labels, breadcrumb, tab labels |
| Label | 12px | 500 (medium) | 1.4 | Toolbar button labels, section labels, error count badge |
| Heading | 16px | 600 (semibold) | 1.25 | Panel titles (if any appear), empty-state heading |
| Code | 14px | 400 (regular) | 1.6 | All CodeMirror editor content — JetBrains Mono |

Font weights in use: 400 (regular) and 600 (semibold). Label uses 500 only for de-emphasis contrast against semibold headings — treat as a sub-weight, not a third family weight.

Source: CONTEXT.md (dark developer-tool aesthetic), Claude's discretion (VS Code / Insomnia reference).

## Color

Dark developer-tool palette. All values are dark-mode only — EDIT-V2-03 (theme toggle) is deferred to v2.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1e1e1e` | App background, editor background, tree panel background |
| Secondary (30%) | `#252526` | Tab bar background, toolbar background, status bar background, sidebar panel surface |
| Accent (10%) | `#0078d4` (VS Code blue) | Active tab indicator underline, selected tree node highlight background, focused border on CodeMirror editor |
| Error / Destructive | `#f44747` | Syntax error squiggles, error badge in status bar, lint gutter markers |
| Success | `#4ec9b0` | Valid JSON indicator in status bar |
| Text Primary | `#d4d4d4` | All body text, tree node labels, tab labels |
| Text Muted | `#858585` | Status bar JSONPath text, inactive tab labels, placeholder text |
| Border | `#3e3e42` | Panel borders, tab dividers, toolbar separator lines |

Accent (`#0078d4`) reserved for:
1. Active tab underline indicator (3px bottom border)
2. Selected tree node row highlight (10% opacity background tint)
3. CodeMirror editor focus ring (1px border)

Accent is NOT used for toolbar buttons, text labels, or hover states.

Source: CONTEXT.md ("dark developer-tool aesthetic consistent with VS Code / Insomnia"), Claude's discretion (VS Code color values).

## Layout Contract

```
┌─────────────────────────────────────────────────────┐
│  App Shell (bg: #1e1e1e, full viewport)             │
│  ┌───────────────────────────────────────────────┐  │
│  │  Tab Bar (h:36px, bg: #252526)                │  │
│  │  [Editor] [Tree] [Transform*]                 │  │
│  │           (* disabled placeholder)            │  │
│  ├───────────────────────────────────────────────┤  │
│  │  Toolbar (h:36px, bg: #252526, border-b)      │  │
│  │  [Open File] [Format] [Minify] [Repair] [Copy]│  │
│  ├───────────────────────────────────────────────┤  │
│  │                                               │  │
│  │  Tab Panel Content (flex-1, overflow hidden)  │  │
│  │  Editor tab: CodeMirror full width/height     │  │
│  │  Tree tab: scrollable tree view               │  │
│  │                                               │  │
│  ├───────────────────────────────────────────────┤  │
│  │  Status Bar (h:28px, bg: #252526, border-t)   │  │
│  │  [JSONPath breadcrumb]       [Error count]    │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

- Tab bar and toolbar are separate rows — toolbar shows context-appropriate actions for the active tab.
- On Tree tab: toolbar shows only Copy (copies selected node value).
- On Editor tab: toolbar shows all five actions.
- Layout uses `height: 100dvh`, flex column. Tab panel takes `flex-1` with `overflow: hidden`.

Source: CONTEXT.md (D-06, D-07), Claude's discretion (tab ordering, toolbar layout).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Format" (verb-only; action is self-evident from context) |
| Empty state heading | "Paste or load JSON to get started" |
| Empty state body | "Paste JSON into the editor, or click Open File to load a .json file from disk." |
| Syntax error badge | "{N} error" / "{N} errors" (pluralized) |
| Valid JSON indicator | "Valid JSON" |
| JSONPath empty state | "Select a node in the Tree view to see its path" |
| Clipboard success | "Copied" (inline button label swap, 1.5s, no toast) |
| File load error | "Could not read file. Make sure it is a valid text file and try again." |
| Repair: nothing to fix | "JSON is already valid — nothing to repair." |
| Transform tab tooltip (disabled) | "Transform with jq — available in the next phase" |

Destructive actions in Phase 1: None. Editing replaces content in the editor directly — no confirmation dialogs required.

Source: CONTEXT.md (success criteria), Claude's discretion (copy tone: direct, lowercase, no marketing language).
