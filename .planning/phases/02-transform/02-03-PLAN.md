<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AppShell → TransformPanel (rawJson prop) | rawJson originates from the user (paste/file input); passed down through React props as a string. |
| TransformPanel → AppShell (onOutputChange) | jq output flows upward as a notification; AppShell stores it in `transformOutput` state. Used only for Copy routing. |
| AppShell → Toolbar (outputText prop) | Transform output flows into Toolbar as a string or null; used only in `handleCopy` (no eval, no innerHTML). |
| Toolbar → Clipboard API | `writeToClipboard` (Plan 01) writes text via `navigator.clipboard.writeText`. No HTML / rich text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | Copy button leaking the wrong content (RESEARCH.md Pitfall 4) | mitigate | `handleCopy` explicitly branches on `activeTab === 'transform'` and copies `outputText ?? ''` — not the editor's `rawJson`. Unit-tested by new Tests A/B/C in Toolbar.test.tsx. |
| T-02-03-02 | Input Validation (V5) | AppShell forwards user-typed rawJson into TransformPanel | accept | rawJson is already untrusted input bounded to the tool's scope — jq evaluates it inside the Worker sandbox. Phase 1 threat model covers this boundary. |
| T-02-03-03 | Denial of Service | Worker spawned eagerly on app load via TransformPanel mount | accept | TransformPanel mounts inside the `value="transform"` TabsContent, which is rendered only when activeTab === 'transform' (per Radix Tabs default). Worker starts on first tab visit — small one-time startup cost, not a DoS vector. |
| T-02-03-04 | Spoofing | Disabled-tab tooltip removed; new enabled tab could confuse users during WASM load | mitigate | Loading state is explicit: OutputPane shows "jq engine loading — Run will be available shortly" and the Run button label reads "Loading…" while `engineReady === false`. Users are not left guessing. Verified in human-verify step 4. |
| T-02-03-05 | Tampering | Production build missing `jq.wasm` (Pitfall 1) | mitigate | `npm run build` now runs `npm run copy-wasm` first (Plan 02-01 Task 1). Acceptance criteria include `test -f dist/jq.wasm`. Human-verify step 14 checks 200 OK in production-preview. |
| T-02-03-06 | Repudiation | No audit log of jq expressions run | accept | This is a client-side dev tool with no persistence. No logging required. |

ASVS L1: V5 coverage via upstream boundary (jq-web sandbox). No new controls introduced by the wiring layer.
</threat_model>
