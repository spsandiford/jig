## Interaction Contract

### Run Button States

| State | Appearance | Interaction |
|-------|-----------|-------------|
| Engine loading | Disabled; `Loader2` spinner + "Loading…" label; `#858585` text | Non-interactive |
| Engine ready, expression empty | Disabled; opacity-40 | Non-interactive |
| Engine ready, expression non-empty | Enabled; `#0078d4` background, `#d4d4d4` text | Click or Ctrl+Enter triggers run |
| Running (async) | Disabled during execution; spinner replaces icon | Prevent double-submission |

### Expression Input

- Multiline `<textarea>`, resize-none, full-width, min-height 80px (5 rows)
- `font-family: ui-monospace, 'Geist Mono', monospace`, `font-size: 13px`, `line-height: 1.5`
- Background: `#252526`, text: `#d4d4d4`, border: `#3e3e42`, border-radius: 4px
- Placeholder text: `.` (the identity jq expression) in `#858585`
- Ctrl+Enter keyboard shortcut fires Run (same as Run button click)
- No auto-run / debounce (D-03: explicit run only)

### Output Pane

- Scrollable, `overflow-auto`, flex-1 (fills remaining height below expression input)
- Background: `#1e1e1e`, text: `#d4d4d4`
- Content is a `<pre>` with `font-family: ui-monospace`, `font-size: 13px`, `white-space: pre-wrap`
- When output is valid JSON: display as formatted JSON string (JSON.stringify with 2-space indent)
- Empty state (before first run): display muted hint text — see Copywriting Contract

### Error Banner (replaces output on jq failure)

- Full-width, background `#1e1e1e`, left border 3px solid `#f44747`
- Padding: 12px 16px (py-3 px-4)
- Error heading: "Expression error" in `#f44747`, 12px, weight 600
- Error body: human-readable message (strip jq internal stack prefixes), `#d4d4d4`, 12px, weight 400
- Last successful output is NOT retained — pane shows only the error state (D-04)

### Engine Loading Indicator (XFRM-02)

- Positioned below the Run button, visible only while engine is initialising
- `Loader2` icon (14×14px, animate-spin) + text: "jq engine loading…"
- Text color: `#858585`, font-size: 11px
- Disappears once engine signals ready; Run button becomes enabled

### Toolbar Behaviour on Transform Tab

- Open File button: remains active (user may load new JSON while on Transform tab)
- Format / Minify / Repair group: hidden (already conditional on `activeTab === 'editor'`)
- Copy button: copies the current output pane text to clipboard (not the input JSON)


Vertical padding between expression area and output pane: 0 — use a 1px border separator only (matches existing `border-[#3e3e42]` language).
