## Color

Dark developer-tool palette. Dark-mode only (EDIT-V2-03 theme toggle deferred to v2).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1e1e1e` | Compare panel background, Left and Right editor pane backgrounds |
| Secondary (30%) | `#252526` | Compare toolbar row background, pane header label strip, mode toggle container |
| Accent (10%) | `#0078d4` | Compare button (enabled state), active tab underline, active mode pill background |
| Error / parse failure | `#f44747` | Invalid JSON banner on Left or Right pane when parse fails |
| Text primary | `#d4d4d4` | Pane header labels, button labels, editor content |
| Text muted | `#858585` | Placeholder text in empty panes, disabled Compare button label, inactive mode pill label |
| Border | `#3e3e42` | Vertical splitter between Left/Right panes, pane header bottom border, toolbar separator |
| Diff removed (Left) | `#4b1818` | Background highlight on Left pane lines where value differs (dark red — readable on `#1e1e1e`) |
| Diff added (Right) | `#1a3a1a` | Background highlight on Right pane lines where value differs (dark green — readable on `#1e1e1e`) |
| Diff changed (both) | `#3a2a00` | Background highlight where value exists in both sides but differs (dark amber) |
| Diff gutter marker (Left) | `#f44747` | Left gutter indicator dot for removed/changed lines |
| Diff gutter marker (Right) | `#4ec9b0` | Right gutter indicator dot for added/changed lines |

Accent (`#0078d4`) reserved for:
1. Compare button background (enabled, primary filled variant)
2. Active tab underline indicator on the Compare tab (inherited AppShell pattern)
3. Active mode pill background in the Value / Structure toggle

Diff colors (`#4b1818`, `#1a3a1a`, `#3a2a00`) are reserved exclusively for inline CodeMirror diff decorations. They are not used for any other UI elements.

Destructive / error (`#f44747`) reserved for:
- Parse error banner that replaces pane content when JSON is invalid in a given pane
- Left pane gutter markers for removed/changed lines

Source: CONTEXT.md D-03 (inline highlights, removed=red/added=green/changed=amber), established dark theme tokens from Phase 1/2 (pre-populated); specific hex values for diff highlights are Claude's discretion calibrated to VS Code dark theme.

## Interaction Contract

### Compare Button States

| State | Appearance | Interaction |
|-------|-----------|-------------|
| Both panes empty | Disabled; opacity-40, text `#858585` | Non-interactive; tooltip: "Paste or load JSON into both panes" |
| One pane empty | Disabled; opacity-40, text `#858585` | Non-interactive; tooltip: "Both panes need JSON to compare" |
| Both panes have valid JSON | Enabled; bg `#0078d4`, text `#d4d4d4` | Click triggers diff computation |
| Either pane has invalid JSON | Disabled; opacity-40 | Non-interactive; ParseErrorBanner visible in affected pane |
| Diff active (after Compare click) | Replaced by Reset button (ghost variant, label "Reset") | Compare button hides; Reset button appears |

### Mode Toggle States

| State | Appearance |
|-------|-----------|
| Value mode active | "Value" pill: bg `#0078d4`, text `#d4d4d4`; "Structure" pill: bg transparent, text `#858585` |
| Structure mode active | "Structure" pill: bg `#0078d4`, text `#d4d4d4`; "Value" pill: bg transparent, text `#858585` |
| Switching mode | Re-runs diff immediately; no intermediate loading state (computation is synchronous on main thread for typical documents) |

### Pane Editing States

| State | Pane Behavior |
|-------|--------------|
| Before Compare | Panes are editable CodeMirror editors; no diff decorations |
| After Compare | Panes become read-only (CodeMirror `readOnly: true`); diff decorations applied |
| After Reset | Panes return to editable; diff decorations cleared; Compare button re-enabled |

### Diff Highlights

| Diff Type | Left Pane | Right Pane |
|-----------|-----------|------------|
| Value changed (same key, different value) | Line bg `#3a2a00`; gutter dot `#f44747` | Line bg `#3a2a00`; gutter dot `#4ec9b0` |
| Key removed (exists in Left, absent in Right) | Line bg `#4b1818`; gutter dot `#f44747` | No highlight |
| Key added (absent in Left, exists in Right) | No highlight | Line bg `#1a3a1a`; gutter dot `#4ec9b0` |

Structure Diff mode suppresses value-change highlights; shows only key-existence differences (Key removed / Key added rows above).

Array diff: highlight entire array element lines by index — no character-level array diff in v1. Array element present in one side and absent in the other is treated as added/removed at that index.

jsondiffpatch execution: runs on the main thread (synchronous). Worker Facade from Phase 2 is not extended for this use case — jsondiffpatch computation is fast enough for typical JSON documents and the delta output must be immediately traversed in the React render cycle to produce decoration ranges.

Source: CONTEXT.md D-03, D-04, D-07, D-08; Claude's discretion (main thread vs worker, array diff granularity, diff color values).

### Open File in Pane Header

- Each pane's header has a `FolderOpen` icon button (16px, icon-only)
- Clicking opens a file picker scoped to that pane only
- When a file is loaded into an active-diff pane, Reset is triggered automatically (diff clears, editing re-enabled)
- Tooltip: "Load JSON file into this pane"

### Keyboard

| Shortcut | Action |
|----------|--------|
| No global shortcut for Compare in v1 | Compare is triggered by button click only |
| Ctrl+A (within pane) | Select all in that pane (default CodeMirror behavior) |
